# Contributing

## Scope

This repository accepts contributions focused on practical security risks in:

- AI-generated code
- Agent workflow/runtime behavior
- MCP security boundaries
- AI DevSecOps and cloud automation

Out-of-scope content (e.g., general AI ethics or non-technical discussion) may be closed.

## Contribution Standards

- Be evidence-driven: include concrete examples, logs, code snippets, or reproducible scenarios.
- Be explicit about assumptions and threat model boundaries.
- Prefer actionable controls over abstract recommendations.
- Keep content technical, concise, and implementation-oriented.

## Pull Request Expectations

- Clear problem statement.
- Security impact analysis.
- Reproduction steps or validation approach.
- Proposed mitigations and operational tradeoffs.

## Style

- Avoid marketing language and broad claims.
- Use precise security terminology.
- Cite primary references when possible.
